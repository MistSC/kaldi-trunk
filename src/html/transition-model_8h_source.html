36 // The class TransitionModel is a repository for the transition probabilities.
37 // It also handles certain integer mappings.
38 // The basic model is as follows. Each phone has a HMM topology defined in
39 // hmm-topology.h. Each HMM-state of each of these phones has a number of
40 // transitions (and final-probs) out of it. Each HMM-state defined in the
41 // HmmTopology class has an associated "pdf_class". This gets replaced with
42 // an actual pdf-id via the tree. The transition model associates the
43 // transition probs with the (phone, HMM-state, pdf-id). We associate with
44 // each such triple a transition-state. Each
45 // transition-state has a number of associated probabilities to estimate;
46 // this depends on the number of transitions/final-probs in the topology for
47 // that (phone, HMM-state). Each probability has an associated transition-index.
48 // We associate with each (transition-state, transition-index) a unique transition-id.
49 // Each individual probability estimated by the transition-model is asociated with a
50 // transition-id.
51 //
52 // List of the various types of quantity referred to here and what they mean:
53 // phone: a phone index (1, 2, 3 ...)
54 // HMM-state: a number (0, 1, 2...) that indexes TopologyEntry (see hmm-topology.h)
55 // pdf-id: a number output by the Compute function of ContextDependency (it
56 // indexes pdf's). Zero-based.
57 // transition-state: the states for which we estimate transition probabilities for transitions
58 // out of them. In some topologies, will map one-to-one with pdf-ids.
59 // One-based, since it appears on FSTs.
60 // transition-index: identifier of a transition (or final-prob) in the HMM. Indexes the
61 // "transitions" vector in HmmTopology::HmmState. [if it is out of range,
62 // equal to transitions.size(), it refers to the final-prob.]
63 // Zero-based.
64 // transition-id: identifier of a unique parameter of the TransitionModel.
65 // Associated with a (transition-state, transition-index) pair.
66 // One-based, since it appears on FSTs.
67 //
68 // List of the possible mappings TransitionModel can do:
69 // (phone, HMM-state, pdf-id) -> transition-state
70 // (transition-state, transition-index) -> transition-id
71 // Reverse mappings:
72 // transition-id -> transition-state
73 // transition-id -> transition-index
74 // transition-state -> phone
75 // transition-state -> HMM-state
76 // transition-state -> pdf-id
77 //
78 // The main things the TransitionModel object can do are:
79 // Get initialized (need ContextDependency and HmmTopology objects).
80 // Read/write.
81 // Update [given a vector of counts indexed by transition-id].
82 // Do the various integer mappings mentioned above.
83 // Get the probability (or log-probability) associated with a particular transition-id.
