37  // which must have no output transitions and must be nonemitting, has

39  // probability; you can treat the transition probability to the final

69 // kNoPdf is used where pdf_class or pdf would be used, to indicate,
70 // none is there. Mainly useful in skippable models, but also used
71 // for end states.
72 // A caveat with nonemitting states is that their out-transitions
73 // are not trainable, due to technical issues with the way
74 // we decided to accumulate the stats. Any transitions arising from (*)
75 // HMM states with "kNoPdf" as the label are second-class transitions,
76 // They do not have "transition-states" or "transition-ids" associated
77 // with them. They are used to create the FST version of the
78 // HMMs, where they lead to epsilon arcs.
79 // (*) "arising from" is a bit of a technical term here, due to the way
80 // (if reorder == true), we put the transition-id associated with the
81 // outward arcs of the state, on the input transition to the state.
