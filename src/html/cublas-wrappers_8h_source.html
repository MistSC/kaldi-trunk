150 // Use caution with these, the 'transpose' argument is the opposite of what it
151 // should really be, due to CUDA storing things in column major order. We also
152 // had to switch 'l' to 'u'; we view our packed matrices as lower-triangular,
153 // row-by-row, but CUDA views the same layout as upper-triangular,
154 // column-by-column.
